CV Model Deployment Platform

API-based platform for deploying and running computer vision models

Platform Overview

This platform allows you to:

  • Upload custom CV models (predict.py + requirements.txt)
  • Automatically build Docker images from your code
  • Run inference jobs on uploaded images
  • Retrieve processed results via presigned URLs

Quick Start Guide

Note: This is an API-based platform. Use curl or any HTTP client to interact with the endpoints. API Base URL: http://localhost:8000

1. Register a User

curl -X POST http://localhost:8000/api/users/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "demo",
    "email": "demo@example.com",
    "password": "password123"
  }'

2. Login to Get Token

curl -X POST "http://localhost:8000/api/users/login?username=demo&password=password123"

Save the access_token from the response.

3. Create a Model

curl -X POST http://localhost:8000/api/models/ \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Grayscale Converter",
    "description": "Converts images to grayscale",
    "is_public": true
  }'

4. Create Model Version & Upload Package

curl -X POST http://localhost:8000/api/models/versions \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model_id": "MODEL_ID",
    "version_number": "v1.0"
  }'

Use the returned upload_url to upload your model zip:

curl -X PUT "PRESIGNED_UPLOAD_URL" \
  --upload-file model.zip

5. Trigger Build

curl -X POST http://localhost:8000/api/models/versions/VERSION_ID/build \
  -H "Authorization: Bearer YOUR_TOKEN"

6. Check Build Status

curl -X GET http://localhost:8000/api/models/versions/VERSION_ID \
  -H "Authorization: Bearer YOUR_TOKEN"

7. Create Inference Job

curl -X POST http://localhost:8000/api/jobs/ \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "version_id": "VERSION_ID"
  }'

8. Upload Input Image

curl -X PUT "JOB_UPLOAD_URL" \
  --upload-file input.png

9. Run Inference

curl -X POST http://localhost:8000/api/jobs/JOB_ID/run \
  -H "Authorization: Bearer YOUR_TOKEN"

10. Get Results

curl -X GET http://localhost:8000/api/jobs/JOB_ID/outputs \
  -H "Authorization: Bearer YOUR_TOKEN"

Download output using the returned presigned URL.

Model Package Format

Your model package should be a ZIP file containing:

  • predict.py - Must define a run(input_path, output_dir) function
  • requirements.txt - Python dependencies
  • Optional: model weights, config files, etc.
Example predict.py: 
from PIL import Image
import os

def run(input_path: str, output_dir: str) -> dict:
    img = Image.open(input_path)
    grayscale = img.convert('L')
    grayscale.save(os.path.join(output_dir, "output.png"))
    return {"status": "success"}

Available Endpoints

  • POST /api/users/register - Register user
  • POST /api/users/login - Login
  • POST /api/models/ - Create model
  • GET /api/models/ - List models
  • POST /api/models/versions - Create version
  • POST /api/models/versions/{id}/build - Trigger build
  • POST /api/jobs/ - Create job
  • POST /api/jobs/{id}/run - Run inference
  • GET /api/jobs/{id}/outputs - Get results
